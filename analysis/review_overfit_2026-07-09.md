## 发现 1（严重）：rot2 "n=31 消失 / n≥33 仅 rct4" 是过度宣称

**声明**：§3.1 "Phase 3 — rot2 solutions vanish at n=31 — a sharp SAT unsatisfiability
transition"；§3.9 "At n=31, rot2 solutions become empirically unsatisfiable — zero
solutions remain" 与 "At n≥33, only rct4 solutions survive in the database"。

**缓存验证**（`validate_readme_claims.py`，扫描 `flammenkamp_cache/`）：

```
rot2 解存在的 n: 4,6,7,...,30, 33   （n=31, n=32 = 0）
n=31 rot2 计数: 0   (§3.10 称 UNSAT → 与缓存一致)
n=32 rot2 计数: 0
n=33 rot2 计数: 1   (n33_rot2.few，已验证为合法 180°对称、2-per-row、无三点共线)
```

**判定**：rot2 在 n=31、n=32 **确实 UNSAT**（缓存佐证 §3.10），但 **n=33 存在一个解**。
因此 §3.1/§3.9 的强断言

> "rot2 在 n=31 消失" / "n≥33 仅 rct4 存活"

**与 §3.10 及缓存自相矛盾**。这正是一个由**窄样本（只看到 n=31 计数归零）推出的相变相变叙事**——
未检查 n=33 即把单点归零当作"通用阈值"。

**连带错误**：§3.1 的 Phase-3 表（L286-290）写 `33 | 0`（rot2），应改为 `33 | 1`；
"经验阈值 ≈74–78.9 约束/变量"（L286-290, L675-680）建立在此错误前提上，应降级为
"n=31,32 UNSAT，但 n=33 有解 → 障碍特定于这些 n，非通用阈值"。

**建议**：§3.1/§3.9 与 §3.10 统一口径——保留"n=31,32 经穷举 UNSAT"这一**已证事实**，
删除"rot2 在 n≥31 通用 UNSAT / n≥33 仅 rct4"的推论，改为"rot2 在 n=33 仍有解，相变叙事不成立"。

## 发现 2（中等）：§3.15 危险超图数字陈旧（来自已修复的 bug 脚本）

**声明**（§3.15 L793）："Forbidden triples are rare and *decrease* with n
(1.16% → 0.80% → 0.48% ...); the main diagonal (1,1) alone carries 743 → 2493 → 6209"。

**验证**：`danger_hypergraph.py` 的 `collinear3` 虽已修正为遍历全部 6 点（见 L82-97 注释），
但 README 引用的 1.16%/0.80%/0.48% 与 743/2493/6209 是**旧 bug 版输出**。重跑修正版：

| n | 禁用三元组占比(修正) | README 写的(陈旧) | (1,1) 危险度(修正) | README 写的 |
|---|---|---|---|---|
| 12 | **12.14%** | 1.16% | **3,261** | 743 |
| 16 | **8.20%** | 0.80% | **10,434** | 2,493 |
| 20 | **5.51%** | 0.48% | **25,057** | 6,209 |

**判定**：
- **定性结论仍成立**：危险度随 n 递减（12%→8%→5.5%）、(1,1) 仍是最危险方向（远超次位
  (3,±1) 的 669/2404/4110，差一个数量级）、解的平均危险度仍低于随机子集（650<750 等）。
  所以 §3.15 的"超图视角"框架本身**未被推翻**。
- **但具体数字全部陈旧且偏低约 10×**，且"rare（1.16%）"在 12% 下谈不上 rare。需更正数字。

**新增内部矛盾**：修正版脚本输出 "known solutions that are independent sets: 0/4"，
而 README L793 声称 "Every sampled solution is a verified independent set in H_n"。
原因：脚本用**宽松超图**（6 点含任一共线即禁），真实解在此口径下并非独立集；
真正的约束是 `d6_dominance_correction.md` 定义的 *genuine* 超边。该句应改为
"在 genuine 超图意义下解为独立集"，或删除"every/verified"的绝对表述。

**建议**：把 §3.15 三组数字替换为修正版值（12.14%/8.20%/5.51%、3261/10434/25057），
措辞由"rare"改为"non-trivial but decreasing"，并修正独立集表述。

## 发现 3（良好）：环人口 4/8/12/16 经全量验证，非过拟合

**声明**（§3.1 L302 / §3.5 L479）："every distance ring ... population is a multiple of 4
(empirically 4, 8, 12, or 16 across all 21,601 rot4 solutions for n=12–72)"。

**验证**（`validate_readme_claims.py`，21,701 个 rot4 解，n=8..72 全覆盖）：
- distinct 环人口值 = **{4, 8, 12, 16}**（精确匹配声明）
- 最大环人口 = 16（出现在 n=54）
- 每个解的所有环人口均为 4 的倍数（Orbit-Ring 定理的实证支撑）

**判定**：✅ 声明完全成立，**非过拟合**。这是 README 中最稳健的实证断言之一。
（注：每解环人口 ≤16 是小 n 规律的外推，但已被 n=54 实测 max=16 支持，目前无反例。）

## 发现 4（注意）：§3.7 回归模型脆弱但已自加 caveat

**声明**：三因子模型 R²=0.834（14点/4参数，adj≈0.78）；平方和模型 R²=0.897。

**问题**：
- 回归的 y（missing %）在 n≤13 为穷举权威值，n≥14 为**估计值**（D₄_miss×比值，见 L563），
  把权威与估计混在同一回归中会**虚高 R² 并引入偏差**——典型的"用同一数据既拟合又验证"。
- 14 个数据点、4 个特征，adj R² 仅 0.78，过拟合风险中等。
- "primality +2.95 主导"在 14 点二值特征上极易过拟合。

**判定**：README 已用 "⚠️ Caveat ... significant risk of overfitting"（L606）与
"exploratory model"（L606）明确标注，未当作定论。**结论本身未被推翻，但系数不应被引用为因果结论**。
值得在评审中保留此 caveat，避免后续被误读为"已证 primality 主导"。

## 发现 5（轻微）：§3.14 "几乎所有 C₄ 解是 [m] 或 [m-1,1]" 措辞过度

**声明**（§3.14 L770）："Almost all C₄ solutions have cycle structure [m] or [m-1,1] ...
these two structures combined account for 35–67% of all solutions"。

**问题**：表内合并占比为 **35–67%**（n=28 仅 35%，即多数解是其他结构），与"almost all"
矛盾；且 `construction_law_final_2026-07-09.md` 显示 **n=58–72 的解多为多圈分解**，
[n]/[m-1,1] 主导性在更大 n 失效。该"law"是 n=12–56 区间经验的**过度泛化**。

**判定**：数据本身真实（限 n=12–56），但"几乎全部"的措辞与单样本外推（n=72 特殊）构成轻度过拟合。
建议改"almost all"为"among n=12–56 solutions, [m] and [m-1,1] are the two most common
structures (35–67% combined)"，并注明大 n 行为待查。
