## 2. 严重问题（事实错误 / 数据误导）— 必须修

### 【C1】论文自相矛盾的数字：rot2 在 n=29 应为 44,828，不是 44,890
- **位置**: `main.tex` 第 489 行 与 第 503 行（同一节内）
- **问题**: 第 489 行写「rot2 … 44,890 solutions exist at n=29, yet zero at n=31」；但同节表格（第 503 行）rot2 在 n=29 明确为 **44,828**。  
  实测缓存：n=29 **Total=44,890**（全类之和），**rot2=44,828**（差 62 = dia1+dia2）。  
  → 正文把"总数"误标成"rot2 类"。表格对、正文错，论文自身矛盾。
- **结论第 602 行**「44,890→72, a 600× collapse」：72 是 n=31 的**总数**（tab:extinct n=31 Total=72），而 44,890 是 n=29 总数——这是**总数→总数**的比较，却被放在"rot2 相变"语境下，标签错误。rot2 的真实崩盘是 **44,828 → 0**。
- **修正**: 正文改「rot2: 44,828 → 0」；或若指总数，明确写「total D₄-inequivalent: 44,890 → 72」。两处统一。

### 【C2】猜想"missing-center 对所有偶数 n≥12 存在"与数据库冲突，且论文漏掉偶数二次消失
- **位置**: `main.tex` 第 339–341 行（Conjecture）；第 314 行标题"even-n threshold"
- **问题**: 缓存显示偶数 missing-center 在 **n=12–30 存在（52/11/103/345/21/54/106/306/534），但 n=32,34,36,…,72 全部为 0**（这些 n 的可用类仅为 rot4/dia，二者按群论必含中心）。
- **更重要**: `README.md` 第 166 行已明确写了「At n≥32, catalogued solutions revert to zero missing-center…」。论文**漏掉了这一对称现象**，使"偶数 n≥12 阈值"读起来像 missing-center 是偶数的永久特征。实际上它和奇数 n≥33 的消失是**平行**的：missing-center 只存在于一个**有限窗口**（偶数 12–30，奇数 7–29 后在已知类消失）。
- **修正**: 把猜想限定为"n≤30 的已知类中存在"，并补一句偶数在 n≥32 的二次消失（与 README 对齐）。

### 【C3】结论"三阶段演化：decline (n=21–29)"——按绝对计数这是错的
- **位置**: `main.tex` 第 601 行、第 358 行叙事
- **问题**: 按绝对 missing-center 计数，n=21→29 是**上升**的：190→234→561→777→2,136（缓存实测），并非 decline。  
  "decline"只对**比率**（7.8%→5.8%→6.2%→4.5%→4.8%，README 第 125 行）成立。论文结论未说明是比率，且论文自己的 tab:d4 **删掉了 n=23,25,28,29**，恰好掩盖了计数回升。
- **修正**: 明确写"decline in *rate*"；并把 n=23,25,28,29 补回表（见 C4），否则叙事与数据脱节。

### 【C4】论文 D₄ 表（tab:d4）比 README 少 5 个已知 n
- **位置**: `main.tex` 第 253–271 行
- **问题**: 缓存与 README 都含 n=23(4003/234)、n=25(9040/561)、n=26(4949/106)、n=28(12203/306)、n=29(44890/2136)，但论文 tab:d4 只列到 n=22 后跳 n=24、n=27、n=30，**漏掉 23/25/26/28/29**。
- **后果**: 与论文第 489/503 行用到的 n=23/25/27/29 rot2 计数自相矛盾（正文引用了 n=29，表却无 n=29）；也造成 C3 的误导。
- **修正**: 补全这 5 行（数据已验证）。
